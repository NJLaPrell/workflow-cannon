# Contributing

Thanks for contributing to Workflow Cannon.

## Before you start

- Read `README.md`, `.ai/PRINCIPLES.md`, and `docs/maintainers/ROADMAP.md`.
- Check `docs/maintainers/TASKS.md` for active or queued execution items.
- Align proposed work with existing task and roadmap direction.

## Contribution flow

1. Open or reference a tracked task/feature.
2. Make focused changes with clear intent.
3. Add or update docs when behavior changes.
4. Run relevant validation checks before proposing merge.
5. Include evidence for behavior changes (command output, screenshots, or logs as applicable).

## Change quality expectations

- Favor small, reviewable diffs.
- Preserve backward compatibility where possible.
- Prefer safe, reversible migrations for state/config changes.
- Avoid introducing undocumented configuration keys or template contracts.

## Pull request guidance

- Explain the why, not just the what.
- Note any operational risk, migration requirement, or rollout consideration.
- Link related feature/task entries.

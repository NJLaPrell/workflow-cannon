# Security Policy

## Reporting a vulnerability

If you discover a security issue, please report it privately to the maintainers.
Do not open a public issue containing exploit details.

## What to include

- A clear description of the issue.
- Affected version or commit.
- Reproduction steps or proof of concept.
- Impact assessment and suggested remediation (if known).

## Response expectations

- Initial acknowledgement target: 3 business days.
- Triage and severity assignment after acknowledgement.
- Coordinated remediation and disclosure timing with reporters.

## Scope highlights

Security-sensitive areas include:

- Credential and secret handling.
- Policy and approval bypass risks.
- Unsafe workspace mutation or injection vectors.
- Data retention, redaction, and privacy controls.

# Task-to-PR Delivery Workflow

1. Select the highest-priority ready task in `.workspace-kit/tasks/state.json`.
2. Implement a scoped change that satisfies the task goal.
3. Run relevant validation checks.
4. Update task status and add evidence notes.
5. Open a PR with why/risk/evidence summary.
6. Route unresolved follow-ups into `.workspace-kit/tasks/state.json` and update `docs/maintainers/ROADMAP.md` context if scope shifts.

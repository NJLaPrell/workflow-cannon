## Purpose

Normative **deterministic** rules: code > CAE for loosening (forbidden); **policy** family vs advisory; **specificity**; **priority**; same-type **merge | shadow | fail**; output shape of **effective bundle**.

## Scope

**In:** Algorithm prose, conflict taxonomy, worked examples, failure modes.  
**Out:** TypeScript implementation (T860).

## Deliverable artifacts

1. **Spec doc** `.ai/cae/precedence-merge.md` (or ADR section) with stepwise evaluation order.
2. **Worked examples** (minimum 5): (a) policy beats think, (b) specificity tie-break, (c) equal-priority conflict → shadow, (d) equal-priority conflict → fail (explicit), (e) disabled activation ignored.
3. **Bundle normative schema** — finalize [`../artifacts/stub-effective-activation-bundle.schema.json`](../artifacts/stub-effective-activation-bundle.schema.json) into `schemas/cae/effective-activation-bundle.v1.json`.

## Decisions to resolve

- Formal **specificity metric** (command exact > prefix > module > workspace-wide, etc.).
- Behavior when policy activations **contradict** each other (error vs shadow vs precedence sub-rules).
- Whether **merge** is commutative and provably deterministic (document proof sketch or test obligations).

## Acceptance criteria

- [ ] Two independent implementers would produce identical bundles on given fixtures (T860 golden tests enforce).
- [ ] Every resolution step maps to a **trace event** type (cross-ref T846).

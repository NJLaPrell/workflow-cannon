## Purpose

Specify behavior when registry invalid, context incomplete, evaluator throws, or storage errors: **safe defaults**, doctor signals, operator recovery commands.

## Scope

**In:** Error taxonomy (`code` strings), fail-open vs fail-closed per surface (advisory vs enforcement), circuit breaker optional.  
**Out:** Code in T858–T866.

## Deliverable artifacts

1. **Doc** `.ai/cae/failure-recovery.md` with matrix: failure × surface × behavior.
2. **Doctor lines** spec: what `wk doctor` should print when CAE misconfigured.
3. **Recovery playbook** section: validate registry, clear cache, disable flag.

## Decisions to resolve

- On CAE error during **enforcement** (post-T866): block command vs fall back to code-only (likely block if enforcement enabled).
- On CAE error during **advisory**: always continue + surface warning.

## Acceptance criteria

- [ ] **Never** bypass code-level safety because CAE failed.
- [ ] Stable `code` strings listed for CLI remediation mapping later.

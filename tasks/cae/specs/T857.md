## Purpose

Produce **initial registry entries** for real repo artifacts (`.ai/playbooks/`, key runbooks, policy docs) with stable IDs — **curated** first tranche, not necessarily exhaustive.

## Scope

**In:** Inventory source file + checked-in registry JSON/YAML per T839; review checklist.  
**Out:** Activation definitions (T840) — may reference artifact ids only.

## Deliverable artifacts

1. **Registry file** at a **PR-reviewable repo path** (e.g. **`registry/cae/artifacts.v1.json`** or **`.ai/cae/registry/artifacts.v1.json`**). Avoid `.workspace-kit/` for content that should be versioned like code unless ADR explicitly places generated/cache data there.
2. **`tasks/cae/artifacts/inventory-notes.md`** — scope, omissions, owners.
3. **PR checklist** for adding new artifacts (id, type, path, review).

## Decisions to resolve

- First tranche depth (only `.ai/**` vs include `docs/maintainers/playbooks/`).
- Naming convention for ids (`ai.playbooks.task-to-phase-branch` style).

## Acceptance criteria

- [ ] Every entry validates against registry schema from T839/T840 chain.
- [ ] No broken repo-relative paths.

## Implementation notes

Use `stub-registry-entry.schema.json` as starting point; replace when normative schema lands.

## Purpose

Define **`traceId`**, ordered **trace events**, linkage to inputs (hashes), and **`explain`** payload levels (summary vs verbose) including redaction.

## Scope

**In:** Trace schema, explanation JSON shape, truncation limits, stability promises across patch versions (TBD).  
**Out:** CLI handlers (T862), storage (T867).

## Deliverable artifacts

1. **Normative schema** evolving [`../artifacts/stub-trace-event.schema.json`](../artifacts/stub-trace-event.schema.json) → `schemas/cae/trace.v1.json` (array + envelope).
2. **`explain` response schema** `schemas/cae/explain-response.v1.json`.
3. **Event taxonomy table**: `eventType | when emitted | payload keys`.

## Decisions to resolve

- Max trace size / truncation behavior.

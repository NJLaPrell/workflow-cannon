## Purpose

Pin **where** CAE runs: task resume/start vs **command invoke**; ordering vs **policy approval** checks; **merge task-level + command-level** activations; caching; failure behavior.

## Scope

**In:** Sequence diagram, call graph, config flags, performance guardrails, “no scattered per-module CAE calls” rule.  
**Out:** Code (T864).

## Deliverable artifacts

1. **Design doc** with ordered steps 1–9 from program context mapped to **functions / files** (proposed).
2. **Cache key** spec: `(taskId?, commandName, registryHash, contextHash)` — invariants when cache invalidates.
3. **Degradation matrix** (T853 overlap): link to failure behaviors.

## Decisions to resolve

- Hooks on **all** `wk run` vs only `run <module-command>`.
- Whether **task-level** evaluation requires active task id from context or is no-op.
- Sync vs async; max latency budget.

## Acceptance criteria

- [ ] Single **orchestrator** module owns merge (no N copies in each module).
- [ ] Clear answer: when is command **blocked** vs **advisory-only** (before T866).

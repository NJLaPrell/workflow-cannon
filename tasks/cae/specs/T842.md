# T842 — Evaluation context contract v1

| Field | Value |
| --- | --- |
| **Task ID** | T842 |
| **Engine `dependsOn`** | T837 |
| **Program context** | [`../CAE-PROGRAM-CONTEXT.md`](../CAE-PROGRAM-CONTEXT.md) |

## Purpose

Define **typed, bounded** inputs to CAE: task slice, command slice, workspace slice, governance slice, queue/operational slice; **nullable reserved** fields for future map signals; forbid opaque “workflow blob.”

## Scope

**In:** Field list, types, sources of truth, redaction rules, canonical serialization for hashing.  
**Out:** SQL queries / builder implementation (T859).

## Deliverable artifacts

1. **Contract doc** with table: `field | type | source | required | redaction`.
2. **TypeScript interface or JSON Schema** `schemas/cae/evaluation-context.v1.json` (choose one as normative; if TS-first, export schema from types in T859).
3. **Negative examples**: oversized/dumped state payloads labeled **non-compliant**.

## Decisions to resolve

- Allowed **task metadata** keys (allowlist) vs “any string” rejected.
- Whether governance comes only from env (`POLICY-APPROVAL` lanes) + config, or also from manifest.
- **Canonical JSON** ordering for deterministic `bundleId` / `traceId` inputs.

## Acceptance criteria

- [ ] Implementer can build context from kit without inventing fields.
- [ ] Future `mapSignals: null` (or omitted) documented.

## Implementation notes

Prefer reading task state via **existing module/task-engine APIs** over ad hoc SQLite unless ADR permits.
